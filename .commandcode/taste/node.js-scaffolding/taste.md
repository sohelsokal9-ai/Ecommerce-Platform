# Node.js Scaffolding
- When scaffolding a Node.js project, follow this order: 1) Create ALL folder structure inside a `src/` directory (src/controllers, src/services, src/models, src/routes, src/types, src/validators, src/config, src/middlewares, src/utils) even if empty, 2) Run npm install commands to get latest package versions (do NOT fabricate/hardcode versions in package.json), 3) Create files: errorHandler, asyncHandler, getEnv utility, env.config, http.config, database.config, bcrypt utility, app-error, index.ts with health API route, nodemon.json, tsconfig.json, and .gitignore. Confidence: 0.90
- When adding authentication to a Node.js project, use JWT strategy with passport-jwt + jsonwebtoken (NOT passport-local + express-session). Install: passport, passport-jwt, jsonwebtoken and their @types. Extract JWT from cookies (req.cookies.accessToken) using ExtractJwt.fromExtractors. Include: passport.config with JwtStrategy, cookie.ts with setJwtAuthCookie/clearJwtAuthCookie, passport initialization in index.ts, user model, user.service.ts, auth.service.ts, auth.controller with login/register/logout/auth status endpoints, auth.route.ts, and import routes in index.ts. Confidence: 0.85
- For app-error.ts, include: ErrorCodes const with ERR_INTERNAL, ERR_BAD_REQUEST, ERR_UNAUTHORIZED, ERR_FORBIDDEN, ERR_NOT_FOUND; ErrorCodeType; AppError base class; and exception classes InternalServerException, NotFoundException, BadRequestException, UnauthorizedException. Confidence: 0.75
- Place asyncHandler and errorHandler in the middlewares/ folder (as asyncHandler.middleware.ts and errorHandler.middleware.ts), not in utils/. Confidence: 0.70
- Do NOT include database connection setup (database.config, connectDatabase in index.ts, MONGO_URI in env.config or .env) in the initial scaffold. Only add MongoDB/mongoose configuration when explicitly asked to "configure database mongodb". Confidence: 0.85
- After setting up a Node.js project, always start the server (e.g., `npm run dev`) to verify it runs, rather than only running static checks like `tsc --noEmit`. Confidence: 0.85
- Always include ZodError handling in the errorHandler middleware: check `if (error instanceof ZodError)` and return a formatted response with the error code `ERR_Validation`. Use a `formatZodError` helper to make the validation errors readable. Confidence: 0.80
